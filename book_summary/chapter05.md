5.2.5.1
ミューテックスロックとセマフォの関係について
わかりにくかったので、整理


ミューテックスロック
排他制御の仕組みの一つ
Thread A: [ロック取得] → 処理中 → [解放]
Thread B:    ← ブロック（待機）→  [ロック取得] → 処理
Thread C:         ← ブロック（待機）→

セマフォ
任意の数を制御する仕組み
Thread A: [取得 → 残2] 処理中
Thread B: [取得 → 残1] 処理中
Thread C: [取得 → 残0] 処理中
Thread D:  ← ブロック（カウンタ=0）

上記で整理すると、書籍中の「一つのスレッドの実行だけを認めるか（この場合、実質的にミューテックスロックになる）」が理解できる。
