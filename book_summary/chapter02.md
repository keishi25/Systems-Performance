# Chapter 02: 深掘りメモ

## 2.5.9.6 ミューテックスロック
- プロセス: 1アプリの実行単位。
- スレッド: 実行の最小単位。プロセス内のスレッド間ではメモリを共有する。
- ミューテックス: 共有メモリへのアクセスを排他制御する仕組み。

OSスレッド（ソフトウェアスレッド）とCPUスレッド（ハードウェアスレッド）は別物。
CPUスレッドは、独立した実行状態を持つ最小単位。

参考リンク:
https://www.nec-lavie.jp/products/contents/core_thread_pc.html

## 2.5.12.1 メモリのページング
メモリを固定サイズ（ページ）に分割し、使っていないページをディスクのswap領域に退避し、必要に応じて戻す仕組み。

CPU使用率が低いのにレイテンシが悪い場合、ページング多発の可能性がある。
確認コマンド: `vmstat`

## 2.5.18 キャッシュの配置
「処理が実行される場所に近い位置でキャッシング」を深掘り。

実行される場所の近くにデータを配置する。

近くにする方法:
1. プロセス内: メモリ共有できるため、辞書型（JavaのMap、Pythonのdict）で保持。
2. プロセス外: メモリ共有できないため、キャッシュ用DB（Redis）を利用。

CPUレベルにはCPUキャッシュがあり、ユーザー側で操作できない。
チップ設計側（Intel、Apple、Nvidiaなど）が意識する領域。

## 2.7.1 ファイルディスクリプタ
プロセスが開いているファイルを識別するための番号（例: クロークの番号札）。

UNIXの考え方:
Everything is a file
I/Oはすべて「ファイル風」に扱えるという意味。
共通インターフェース: `open`, `read`, `write`, `close`

ファイルではないもの:
- プロセスのkillコマンド
- メモリの読み取り（mmapコマンド）

## 2.7.2 メモ
最適なリソース選択に使えそう。
パフォーマンスとコストの最適化手法で使えそう。
